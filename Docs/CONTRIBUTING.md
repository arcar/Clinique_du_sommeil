
### Convention de nommage : Nom de branch
```bash
   -> dev
       -> prenom
	-> prise en charge ticket
```

### Convention de nommage : Noms des commits :
```bash
    <Feat> Ajout d'une nouvelle fonctionnalité
    <Fix> Correction des bugs
    <Docs> Mise à jour ou création de documents
    <Chores> Réparation des "boulettes"
```

### Règles obligatoires pour les pull requests:
```bash
• une branche par développeur ;
• une Pull Request par ticket ;
• pas de push direct sur main ;
• pas de push direct sur develop, sauf consigne explicite ;
• besoin de validation avant un merge sur dev
```
